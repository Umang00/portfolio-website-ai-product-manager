| Aspect               | Standard                              |

## Part 1: Resume Anatomy

### 1.1 Section Order (Optimal)

1. **Contact Information + Header**
2. **Professional Summary** (2-3 lines)
3. **Professional Experience** (the core)
4. **Skills** (technical focus)
5. **Projects** (if space permits)
6. **Education**
7. **Certifications** (optional)

### 1.2 Contact Information

**Must Include:**

- Full name (largest text)
- Phone number
- Professional email
- LinkedIn URL (hyperlinked)
- Portfolio website / GitHub
- City, State (no full address)

**Exclude:**

- Photo (US/UK standard)
- Age, marital status, personal details

### 1.3 Professional Summary

**Structure:** Years of experience + Specialization + Key achievement + Core competencies

**Template:**

```
[Role] with [X]+ years driving [domain]. [Key achievement with metric].
Expert in [3-4 relevant competencies aligned to target role].
```

**Example for AI PM:**

```
AI Product Manager with 4+ years driving B2C product growth. Increased user
engagement by 200% through personality-based matching at Hunch. Expert in
prototyping LLM products, user research automation, and data-driven
roadmapping.
```

**For Career Changers:**

```
Data-driven Product Leader with 5 years in Solutions Engineering transitioning
to Product Management. Proven ability to translate customer pain points into
technical requirements. Skilled in SQL, A/B testing, and cross-functional leadership.
```

## Part 2: The Bullet Point Formula

### 2.1 Frameworks Ranked

| Framework       | When to Use                   | Structure                                          |
| --------------- | ----------------------------- | -------------------------------------------------- |
| **XYZ Formula** | Tech companies (Google, Meta) | "Accomplished [X] as measured by [Y] by doing [Z]" |
| **STAR Method** | Traditional corporations      | Situation → Task → Action → Result                 |
| **CAR Method**  | Startups, concise bullets     | Challenge → Action → Result                        |

### 2.2 XYZ Formula (Primary Recommendation)

**Formula:** "Accomplished [X] as measured by [Y] by doing [Z]"

**Examples:**

- "Increased user retention (X) by 25% (Y) by implementing a personalized onboarding experience and gamification features (Z)."
- "Reduced campaign launch time from 2 days to 5 minutes (Y) by designing a self-service targeting interface (Z), eliminating 85% of manual work (X)."

### 2.3 STAR Method (Detailed Version)

Use when space permits and context matters:

```
Situation: Market challenge or product problem
Task: Your specific PM responsibility
Action: Strategic decisions and execution steps
Result: Business impact with metrics
```

**Example:**
"Identified declining user engagement in onboarding flow (S) and was tasked with improving completion rates by 20% (T). Conducted user research and A/B tested three redesigned flows (A), resulting in 35% improvement in completion rates and $500K additional ARR (R)."

### 2.4 The 80% Rule

**80% of bullets should include quantifiable metrics.**

Essential metrics for PMs:

- **User Impact:** Engagement rates, retention, DAU/MAU, satisfaction scores
- **Business Impact:** Revenue, conversion rates, cost savings, ARR
- **Scale Indicators:** Team sizes, user base, feature adoption
- **Efficiency Gains:** Time savings, process improvements, automation rates

### 2.5 Bullet Count by Role

| Role Recency         | Bullet Count |
| -------------------- | ------------ |
| Current role         | 4-5 bullets  |
| Previous roles       | 3-4 bullets  |
| Older roles (5+ yrs) | 2-3 bullets  |

**Quality over quantity.** Better to have 3 exceptional bullets than 6 mediocre ones.

### 2.6 PM-Specific Strong Verbs

| Category   | Verbs                                       |
| ---------- | ------------------------------------------- |
| Strategic  | Defined, Prioritized, Aligned, Optimized    |
| Analytical | Analyzed, Measured, Tested, Validated       |
| Leadership | Led, Coordinated, Collaborated, Influenced  |
| Execution  | Launched, Implemented, Delivered, Scaled    |
| Growth     | Drove, Increased, Boosted, Expanded         |
| Efficiency | Reduced, Automated, Streamlined, Eliminated |

## Part 3: Experience Section Deep Dive

### 3.1 Essential Elements Per Role

```
[Job Title] | [Company Name]
[City, State] | [Month/Year] – [Month/Year]

• [Impact-driven bullet using XYZ framework]
• [Cross-functional leadership example]
• [Technical/analytical achievement]
• [Strategic initiative with business outcome]
```

### 3.2 PM-Specific Content Requirements

Every PM experience section should demonstrate:

1. **Product Lifecycle Ownership** - End-to-end examples
2. **Cross-Functional Leadership** - Specify team sizes and roles
3. **User Research** - Data-driven decision making
4. **Revenue/Growth Impact** - Specific metrics
5. **Technical Collaboration** - Work with engineering
6. **Go-to-Market Strategy** - Launch and positioning

### 3.3 Number Placement Strategy

| Style            | When to Use        | Example                                                |
| ---------------- | ------------------ | ------------------------------------------------------ |
| **Front-loaded** | For maximum impact | "200% increase in user engagement achieved through..." |
| **End-loaded**   | For STAR format    | "...resulting in 200% increase in user engagement"     |
| **Embedded**     | Natural flow       | "Led a team of 12 engineers to deliver..."             |

## Part 4: Skills Section

### 4.1 The Hybrid Approach

Use BOTH:

1. **Standalone Technical Skills Section** (for scannability and ATS)
2. **Integrated Within Experience** (prove the skills with examples)

### 4.2 Skills Categories for AI PMs (2025)

```
Product Management: Roadmapping, PRDs, User Research, A/B Testing, OKRs,
                    Agile/Scrum, Prioritization (RICE/WSJF), Go-to-Market

Technical: SQL, Python, TypeScript, REST APIs, Data Analysis

AI/ML: LLM Fine-tuning, RAG Systems, Prompt Engineering, Evals,
       Vector Databases, Agent Architecture, MCP Servers

Tools: JIRA, Confluence, Figma, Mixpanel, Amplitude, Retool,
       GitHub, Notion, Miro

Cloud: AWS, GCP, Vercel, Supabase
```

### 4.3 Hot Skills for 2025 AI PMs

1. **AI/ML Understanding:** LLM applications, hallucination management
2. **Data Analysis:** Advanced SQL, Python, statistical analysis
3. **Design Collaboration:** Figma, user research methodologies
4. **Growth Metrics:** Retention cohorts, LTV/CAC analysis
5. **Evals:** LLM-as-a-judge, hallucination rate, latency measurement
6. **Agentic AI:** Multi-agent orchestration, tool use, MCP

### 4.4 Soft Skills Rule

**Never list soft skills as bullet points.**

❌ "Strong leadership skills"
✅ "Led cross-functional team of 12 engineers and designers..."

## Part 5: Projects Section

### 5.1 When to Include Projects

**Dedicated Projects Section When:**

- Recent graduate or career switcher
- Freelance PM work or consulting
- Significant side projects with measurable impact
- Hackathon wins or innovation challenges

**Integrated Approach For Experienced PMs:**

- Include notable projects within relevant work experience
- Or add a "Selected Projects" section after Skills

### 5.2 Project Format

```
[Project Name] | [Tech Stack]
• [One-line description with outcome]
• [Key metric or technical achievement]
```

**Example:**

```
AI Food Analyzer | React Native, Gemini 3 Pro, FastAPI
• Mobile app enabling dietary safety checks for allergies and religious restrictions
• Shipped in 15 days; 95%+ accuracy on multilingual ingredient analysis
```

### 5.3 Project Selection for AI PM

Prioritize projects that show:

1. **Business outcome** (revenue, retention, efficiency)
2. **AI/ML product work** (not just technical implementation)
3. **User-facing impact** (shipped to real users)
4. **Cross-functional collaboration** (if applicable)

## Part 7: ATS Optimization

### 7.1 ATS Facts

- 98.4% of Fortune 500 companies use ATS
- Modern ATS handles both PDF and Word
- ATS doesn't assign scores—it's a database recruiters search

### 7.2 ATS-Friendly Formatting

**Do:**

- Simple, sans-serif fonts (Calibri, Helvetica, Lato)
- 10-12pt font size
- Margins no smaller than 0.5 inches
- Clean section headers
- PDF format (default)

**Don't:**

- Columns or tables (breaks parsing)
- Graphics, icons, logos
- Fancy templates (Canva often breaks ATS)
- Headers/footers for critical info

### 7.3 Keyword Strategy

1. **Extract from job description:** Key verbs, nouns, required skills
2. **Mirror exact language:** If they say "orchestrate," use "orchestrated"
3. **Natural integration:** Keywords in Summary + Skills + Experience
4. **Avoid stuffing:** Maintain readability

### 7.4 Recommended Tools

| Tool              | Use Case               |
| ----------------- | ---------------------- |
| **Jobscan**       | Keyword match analysis |
| **Resume Worded** | General optimization   |
| **Teal**          | PM-focused templates   |

## Part 8: Common PM Resume Mistakes

### 8.1 Fatal Errors

1. **Vague Product Impact:** "Worked on product improvements" ❌
   - Fix: "Increased user retention by 25% through onboarding redesign" ✅

2. **Missing Cross-Functional Evidence:** No mention of engineering, design
   - Fix: "Led cross-functional team of 8 engineers, 2 designers..."

3. **Lack of User Focus:** No user research or customer empathy
   - Fix: "Conducted 50+ user interviews to validate..."

4. **No Go-to-Market Experience:** Missing launch strategy
   - Fix: "Launched feature to 100K+ users with 35% adoption rate"

5. **Generic Metrics:** Percentages without context
   - Fix: Add absolute numbers or business context

6. **Process Over Outcome:** Listing methodologies without results
   - Fix: "Used Agile" → "Delivered MVP in 4 weeks using Agile sprints"

7. **Responsibility vs. Achievement:** "Managed roadmap" ❌
   - Fix: "Delivered roadmap that increased revenue by $2M" ✅

8. **Tool Name-Dropping:** Listing tools without proficiency evidence
   - Fix: Integrate tools into achievement bullets

### 8.2 Formatting Mistakes

- Typos or grammatical errors (instant disqualification)
- Inconsistent date formats
- Missing contact information
- Excessive design elements
- Using "responsible for" instead of impact language

## Part 9: Tailoring Strategy

### 9.1 The 80/20 Rule

- **80% core content** remains consistent
- **20% tailored elements** adjusted per application

### 9.2 What to Tailor

1. **Keywords from job description**
2. **Bullet point order** (emphasize relevant experience)
3. **Skills section optimization**
4. **Summary line adjustments**
5. **Project selection**

### 9.3 Tailoring Process

**Step 1: Job Description Analysis**

- Extract 5-8 key requirements and keywords
- Identify must-have vs nice-to-have
- Note company values

**Step 2: Content Mapping**

- Match existing experiences to requirements
- Identify stories demonstrating competencies
- Select achievements for emphasis

**Step 3: Language Alignment**

- Use exact terminology from job posting
- Replace generic terms with company-specific language

## Part 10: Regional Considerations

### 10.1 United States

- Concise, metrics-driven
- One page strongly preferred (mid-level)
- Individual achievements emphasized
- ATS optimization critical

### 10.2 India

- Hybrid approach in tech sector
- 2 pages acceptable
- US-style preferred by multinationals
- Strong emphasis on educational credentials

### 10.3 Europe

- More detailed descriptions accepted
- 2-3 pages standard
- Education/certifications weighted higher
- Country-specific variations (UK closer to US)

## Part 11: The AI PM Differentiator (2025)

### 11.1 What Sets AI PMs Apart

Modern AI PM resumes must demonstrate:

1. **Evals Literacy:** How you measure AI quality (hallucination rate, latency, precision)
2. **Prototyping Ability:** Building functional demos before committing engineering
3. **Trade-off Navigation:** Cost vs. latency vs. quality decisions
4. **Context Engineering:** Understanding prompts, RAG, fine-tuning
5. **Agent Understanding:** Multi-agent, tool use, MCP

### 11.2 Metrics Specific to AI Products

| Metric                     | What It Shows      |
| -------------------------- | ------------------ |
| Hallucination Rate         | Quality control    |
| TTFT (Time to First Token) | Latency management |
| Containment Rate           | Automation success |
| Cost per Query             | Unit economics     |
| Retrieval Precision        | RAG quality        |

### 11.3 AI PM Portfolio Proof Points

Include evidence of:

- Working prototypes (Streamlit, Cursor-built demos)
- GitHub repos with runnable code
- Case studies with before/after metrics
- Technical trade-off documentation

## Part 12: Maintenance Strategy

### 12.1 Update Frequency

| Cadence         | Action                           |
| --------------- | -------------------------------- |
| Monthly         | Update current role achievements |
| Quarterly       | Comprehensive review             |
| Annually        | Complete overhaul                |
| Per Application | 20% customization                |

### 12.2 Achievement Tracking

Maintain a "Quarterly Log" document tracking:

- Quantified results from your work
- Project stories in STAR/XYZ format
- New skills, tools, certifications
- Recognition, promotions, feedback
- Cross-functional wins

### 12.3 Monthly Review Questions

1. What metrics improved under my influence this month?
2. Which cross-functional relationships did I strengthen?
3. What new skills or tools did I master?
4. Which processes did I improve or optimize?
5. What user problems did I solve?
